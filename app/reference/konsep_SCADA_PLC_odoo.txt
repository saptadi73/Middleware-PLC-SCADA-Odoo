Konsep integrasi SCADA - PLC - Odoo (VERSI AKTUAL)

1. Middleware mengambil MO list dari Odoo, simpan ke mo_batch, lalu write ke PLC WRITE area.
2. Saat write, middleware selalu mengisi default status di WRITE:
   - status_manufacturing = 0
   - status_operation = 0
   - status_read_data = 0
3. PLC menjalankan proses produksi berdasarkan data di WRITE area.
4. PLC menulis feedback proses ke READ area (actual consumption, actual weight, status).
5. Middleware membaca READ area periodik dan update actual ke mo_batch.
6. Middleware hanya menandai status_read_data READ area = 1 untuk batch yang sudah selesai
   (status_manufacturing=1 pada payload READ).
7. Jika status_manufacturing di DB sudah 1, Task 2 tidak melakukan update ulang untuk batch tersebut.
8. Batch completed diproses Task 3: sync ke Odoo lalu archive ke mo_histories.
9. Jika sync Odoo gagal atau partial, batch tetap di mo_batch untuk retry; tidak dihapus.
10. Jika status_operation=1 (failure), flow cancel/archive mengikuti logic auto-cancel yang berlaku.

Prinsip ownership status:
- WRITE status proses dimiliki PLC (middleware kirim default 0 saat initial write).
- READ status_read_data dimiliki handshake middleware setelah valid read completed batch.
